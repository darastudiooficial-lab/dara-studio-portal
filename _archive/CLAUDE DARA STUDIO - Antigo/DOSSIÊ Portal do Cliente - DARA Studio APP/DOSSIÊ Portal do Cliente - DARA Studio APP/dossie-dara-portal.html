01Visão Geral e Design

Resumo do Projeto

O Portal do Cliente DARA Studio é uma aplicação SaaS de página única (Single Page Application) construída em React que permite que clientes da DARA Studio acompanhem projetos de arquitetura e design em tempo real, gerenciem faturas e se comuniquem diretamente com a equipe.

Objetivo principal: eliminar a comunicação por e-mail e WhatsApp, centralizando toda a gestão de projeto — arquivos, orçamentos, tickets de revisão e pagamentos — em um único portal seguro e rastreável.

Princípios de UX

  • Otimistic UI: mensagens de chat, site updates e tickets aparecem instantaneamente na tela antes da confirmação do banco — o usuário nunca espera.
  • Feedback de estado: todo botão de ação tem estado Salvando… e ✓ Salvo! para comunicar progresso.
  • Modo demonstração: quando o Supabase não está configurado, o portal funciona com dados fictícios — útil para testes e demos a clientes.
  • Segurança de preço: o orçamento é sempre recalculado pelo backend (serverCalcPrice), jamais pelo valor digitado no browser.
  • Bloqueio por aprovação: campos de detalhes técnicos ficam read-only após o orçamento ser marcado como "Aprovado", exigindo autorização do admin para alterações.

async function fetchProfile(userId) {
  if (!supabase) return DEMO_USER;
  const { data, error } = await supabase
    .from("profiles")
    .select("id, full_name, company, email, phone, avatar_url")
    .eq("id", userId)
    .single();
  if (error || !data) return DEMO_USER;
  return {
    id:       data.id,
    name:     data.full_name,
    company:  data.company,
    email:    data.email,
    phone:    data.phone,
    initials: (data.full_name||"??").split(" ").map(w=>w[0]).join("").slice(0,2).toUpperCase(),
    avatar:   data.avatar_url || null,
  };
}

async function fetchProjects(userId) {
  if (!supabase) return DEMO_PROJECTS;
  const { data, error } = await supabase
    .from("projects")
    .select(`id, code, address, city, service, stage, status,
             budget, paid, progress, updated_at, pending_action`)
    .eq("client_id", userId)
    .order("updated_at", { ascending: false });
  if (error || !data) return DEMO_PROJECTS;
  return data.map(p => ({
    id:            p.id,
    code:          p.code,
    address:       p.address,
    city:          p.city,
    service:       p.service,
    stage:         p.stage,

    paid:          p.paid,
    progress:      p.progress,
    updatedAt:     new Date(p.updated_at).toLocaleDateString("pt-BR",{day:"2-digit",month:"2-digit",year:"numeric"}),
    pendingAction: p.pending_action || null,
  }));
}

Como Ativar o Supabase em Produção

  1. Instale o SDK: npm install @supabase/supabase-js
  2. Crie o arquivo src/lib/supabase.js:
    import { createClient } from '@supabase/supabase-js'
export const supabase = createClient(
  import.meta.env.VITE_SUPABASE_URL,
  import.meta.env.VITE_SUPABASE_ANON_KEY
)
  3. No topo de PortalCliente.jsx, substitua:
    // ANTES:
const SUPABASE_READY = false;
const supabase = null;

// DEPOIS:
import { supabase } from './lib/supabase';
const SUPABASE_READY = true;
  4. Preencha o arquivo .env com suas credenciais reais do Supabase.
  5. Execute npm run build para gerar o bundle de produção.
